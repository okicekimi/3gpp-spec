# 3GPP Specs Archive

Automated archive of 3GPP TS specifications (Rel-15 ~ Rel-19), downloaded from ETSI and converted to Markdown.

## Directory Structure

```
3gpp-specs/
├── urls_specs_download.md   # List of PDF URLs to download
├── urls_specs_for_AI.md     # Auto-generated index of all Markdown files
├── specs-pdf/               # Downloaded PDF files
│   ├── ts_37_213_v1503.pdf
│   ├── ts_38_211_v1806.pdf
│   └── ...
├── specs-md/                # Converted Markdown files
│   ├── ts_37_213_v1503.md
│   ├── ts_38_211_v1806.md
│   └── ...
└── .github/
    └── workflows/
        └── fetch-3gpp-specs.yml    # GitHub Actions workflow
```

## How to Update

Go to **Actions** → **Fetch, Convert and Index 3GPP Specs** → **Run workflow**

| Parameter | Description | Example |
|-----------|-------------|---------|
| `mode` | `missing` only fetches files not yet in the repo; `all` re-downloads everything | `missing` |
| `ts_filter` | Only process a specific spec number, leave empty for all | `38.331` |
| `rel_filter` | Only download a specific release prefix, leave empty for all | `18.` |

## How to Add URLs

Edit `urls_specs_download.md` and add one ETSI PDF URL per line:

```
https://www.etsi.org/deliver/etsi_ts/138200_138299/138211/18.06.00_60/ts_138211v180600p.pdf
https://www.etsi.org/deliver/etsi_ts/138300_138399/138331/18.06.00_60/ts_138331v180600p.pdf
```

Lines starting with `#` and blank lines are ignored.

## Notes

- Source: [ETSI Deliver](https://www.etsi.org/deliver/)
- PDFs are downloaded from ETSI (e.g. `ts_138211v180600p.pdf`) and stored under a shortened name, e.g. `ts_38_211_v1806.pdf`
- Converted Markdown files are stored in `specs-md/` with the same base name, e.g. `ts_38_211_v1806.md`
- A full file index is auto-generated in `urls_specs_for_AI.md` after each run
- Failed download URLs are recorded in `failed_downloads.txt`
